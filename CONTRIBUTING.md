# Contributing

Thanks for your interest in `OpenClaw Neige`.

## Before You Start

- Read `README.md` for the project overview
- Use Node.js `22+`
- Install dependencies with `npm install`

## Development Flow

1. Create a topic branch from `main`
2. Keep changes focused and small
3. Run:
   - `npm test`
   - `npm run typecheck`
4. Open a pull request using the repository template

## Pull Request Expectations

- Explain the problem and the chosen solution
- Call out any config or docs changes
- Add or update tests when behavior changes
- Keep unrelated refactors out of the same PR

## Commit Style

Prefer concise, action-oriented commit messages, for example:

- `feat: add project-state prompt assembly`
- `docs: add GitHub maintainer playbook`
- `ci: add release drafter workflow`

## Reporting Bugs

Please use the Bug Report issue form and include:

- what you expected
- what happened instead
- how to reproduce
- your OpenClaw and Node.js versions

## Proposing Features

Please use the Feature Request issue form and describe:

- the problem you want to solve
- the workflow you want to improve
- why it belongs in Neige rather than a one-off local script

## Security

Do not open public issues for security vulnerabilities. Follow `SECURITY.md` instead.

## Language Policy

- End-user docs should be maintained in English and Simplified Chinese
- Code comments should stay concise and only explain non-obvious logic
